# v3.6.1 发布说明

本次 patch 版本聚焦 **SecretInput 凭据引用、AI Card 生命周期稳定性、多图入站媒体恢复，以及 v3.6.0 用户文档同步**。

**最新版本入口**：[`latest.md`](./latest.md)

## ✨ 功能与体验改进

* **支持 DingTalk `clientSecret` 使用 SecretInput 引用**
  * [`PR #528`](https://github.com/soimy/openclaw-channel-dingtalk/pull/528)（by @cuyua9）新增 `secret-input` 支持，让 `clientSecret` 可以通过宿主 SecretInput 引用读取，避免在配置文件中直接保存明文密钥；同时补齐 setup/onboarding 路径、插件 manifest、配置文档和单元测试。

## 🛠 修复与稳定性

* **修复主动卡片完成时 block 变量收尾问题**
  * [`PR #546`](https://github.com/soimy/openclaw-channel-dingtalk/pull/546)（by @soimy）修复 proactive AI Card finalize 阶段的 block variables 处理，降低卡片最终态内容不完整的风险。

* **修复已打开流式卡片的 finalize 生命周期**
  * [`PR #548`](https://github.com/soimy/openclaw-channel-dingtalk/pull/548)（by @soimy）补齐 opened streaming lifecycle 的关闭与恢复场景测试，避免流式卡片在完成阶段遗留异常状态。

* **防止并发入站消息重复创建 AI Card**
  * [`PR #543`](https://github.com/soimy/openclaw-channel-dingtalk/pull/543)（by @soimy）在卡片创建到分发完成之间增加同步 in-flight guard，避免同一 `accountId:conversationId` 的并发消息绕过 session lock 而重复创建卡片，并补齐异常清理测试。

* **修复富文本多图消息只传递单个媒体路径的问题**
  * [`PR #545`](https://github.com/soimy/openclaw-channel-dingtalk/pull/545)（by @soimy）让多图 rich text 入站消息下载并传递全部 `mediaPaths`，改善多图消息、引用恢复和媒体上下文的完整性。

## 📚 文档

* **同步 v3.6.0 用户文档**
  * [`PR #547`](https://github.com/soimy/openclaw-channel-dingtalk/pull/547)（by @soimy）补齐 AI Card、`/btw`、配置与入门文档中的 v3.6.0 行为说明，让发布页和用户文档保持一致。
