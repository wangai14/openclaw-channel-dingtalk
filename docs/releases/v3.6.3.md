# v3.6.3 发布说明

本次 patch 版本聚焦 **群聊 AI 卡片流式渲染、Markdown 表格兼容性，以及会话媒体回复路由** 的修复。

**最新版本入口**：[`latest.md`](./latest.md)

## 🛠 修复与稳定性

* **修复群聊 AI 卡片流式内容重复渲染问题**
  * [`PR #554`](https://github.com/soimy/openclaw-channel-dingtalk/pull/554)（by @soimy）修复了 AI 卡片在流式输出时内容重复渲染的问题，确保队列内容在卡片边界处正确停止。

* **修复群聊 AI 卡片空白问题**
  * [`PR #557`](https://github.com/soimy/openclaw-channel-dingtalk/pull/557)（by @soimy）恢复了群聊场景下 AI 卡片的流式生命周期初始化调用。此前 #548 的修复移除了初始 streaming PUT 调用，导致群聊卡片无法接收后续变量更新，显示空白并触发 markdown 降级。
  * 新增 `streamLifecycleOpened` 状态持久化，确保崩溃恢复时能正确关闭钉钉侧的流式生命周期。
  * 流式初始化失败不再触发账户级卡片降级（与 `finalizeAICardStreamingLifecycleIfNeeded` 行为对齐）。

* **修复 Markdown 表格在钉钉消息中的渲染问题**
  * [`PR #562`](https://github.com/soimy/openclaw-channel-dingtalk/pull/562)（by @soimy）修复了 Markdown 表格在钉钉消息中无法正确渲染的问题。
  * 放宽表格分隔符识别正则，支持钉钉 `:-:` 格式（仅需 1 个短横线）。
  * 保留原始表格对齐方式（左对齐 `:---`、居中 `:---:`、右对齐 `---:`），而非强制居中。
  * 卡片模式不受影响 —— 钉钉卡片 Markdown 2.0 已正确渲染表格。

* **修复会话媒体回复的路由问题**
  * [`PR #558`](https://github.com/soimy/openclaw-channel-dingtalk/pull/558)（by @soimy）修复了会话场景下媒体回复的路由逻辑，确保媒体消息正确发送到目标会话。
  * 终端卡片媒体回复改为主动发送模式。
  * 会话图片回复在 markdown 模式下合并发送。
  * 媒体上传失败时正确降级处理。

* **修复会话回复中 Markdown 结尾内容丢失问题**
  * [`PR #565`](https://github.com/soimy/openclaw-channel-dingtalk/pull/565)（by @soimy）确保会话回复场景下 Markdown 消息的结尾内容被正确保留。

## 📚 文档

* **同步用户文档更新**
  * [`PR #566`](https://github.com/soimy/openclaw-channel-dingtalk/pull/566)（by @soimy）同步了 PR #553 和 PR #565 的用户文档更新。
