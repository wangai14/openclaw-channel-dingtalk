# 消息类型支持

本页汇总插件支持的入站与出站消息能力，以及常见限制。

## 接收能力

| 类型 | 支持 | 说明 |
| --- | --- | --- |
| 文本 | 是 | 完整支持 |
| 富文本 | 是 | 提取文本内容 |
| 图片 | 是 | 下载并传给 AI |
| 语音 | 是 | 优先使用钉钉识别结果 |
| 视频 | 是 | 下载并传给 AI |
| 文件 | 是 | 下载并传给 AI；常见文本附件会尝试抽取正文 |
| 钉钉文档/钉盘文件卡片 | 是 | 解析卡片中的文件信息后按文件消息处理 |
| 引用文字 | 是 | 提取引用内容作为上下文前缀 |
| 引用图片 | 是 | 使用回调自带的下载句柄恢复 |
| 引用图文 | 是 | 解析 rich text 和引用图片 |
| 引用文件/视频/语音 | 是 | 单聊优先精确恢复；群聊存在兜底链路限制 |
| 引用 AI 卡片 | 是 | 仅支持机器人自己发送的 AI 卡片 |

## 接收侧注意事项

- 附件正文抽取只对较小文本类文件做固定策略处理
- 群聊首次恢复文件类引用时，可能受时间窗口、分页范围和企业认证限制影响
- 图片与图文引用不依赖机器人此前是否见过原消息

## 发送能力

| 类型 | 支持 | 说明 |
| --- | --- | --- |
| 文本 | 是 | 完整支持 |
| Markdown | 是 | 支持自动检测和发送 |
| AI 卡片 | 是 | 支持流式更新 |
| 图片 | 是 | 先上传再发送 |
| 语音 | 是 | 先上传再发送 |
| 视频 | 是 | 先上传再发送 |
| 文件 | 是 | 先上传再发送 |
| 原生语音消息 | 是 | `asVoice=true` 可用 |

## 当前限制

- 不支持图片与 Markdown 或 AI 卡片正文的图文混排
- 远程媒体 URL 下载默认带超时、大小限制与 SSRF 防护
- `asVoice=true` 仍需要真实音频文件，不会自动把纯文本转语音
